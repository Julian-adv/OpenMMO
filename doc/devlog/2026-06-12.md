# Devlog - 2026-06-12

## NPC Source of Truth Unification

NPC definitions were split across two places: `data-src/npcs.csv` (game data, in git) and `agent-client/data/config.toml` (deployment config, gitignored — the real file lives only on the remote host). Karl's name, class and prompt paths were spelled out in both. Default NPCs like Rica/Karl are part of the game, so who they *are* now lives only in git; the deployment file only decides what runs where.

`npcs.csv` gained a `class` column (prompt template + character class), and a `[[npcs]]` config entry now just references the registry by `id` — name, class and the `data/npcs/{id}/` prompt/schedule files are derived at startup, with explicit fields as overrides. Deployment config keeps only what's per-instance: which NPCs to run, account, LLM backend, timing.
